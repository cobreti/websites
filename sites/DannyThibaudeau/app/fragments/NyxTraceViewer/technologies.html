Architecture

This application architecture is bases on an home made cross-platform C/C++ Layer. This layer is independant of any existing framework and rely only on standard C++ libraries (STL) and operating system calls. This gives more possibilities on which platform it can be ported, and gives more flexibility for fine tuning on each platform.

User Interface

Qt was used to make the GUI at first 'cause I intended to get the application on all three platforms as fast as possible. Plans are (if time allows) to build a Cocoa, Java and .Net version of the GUI side to access new platforms where Qt isn't available, as well as for a learning experience. Although the GUI might change, the core module responsible for receiving and managing traces will remain in native C++ for performance reasons.
